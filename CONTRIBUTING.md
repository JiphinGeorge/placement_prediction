# Contributing Guidelines

Thank you for considering contributing to the Student Placement Prediction project!

## How to contribute

1. Fork the repository and create a new branch for your feature or bug fix.
2. Make your changes, ensuring code is well-formatted and tested.
3. Commit with a clear message and push to your fork.
4. Open a pull request detailing the purpose of your changes.

## Reporting issues

- Use GitHub Issues to report bugs or request enhancements.
- Provide a clear description and steps to reproduce the problem.

## Code style

- Follow standard Python conventions (PEP 8).
- Keep dependencies minimal and add any new requirements to `requirements.txt` if present.

## Testing

- Add tests for new features where appropriate.
- Ensure existing functionality is not broken.

## Code of conduct

Please be respectful and follow the [Contributor Covenant](https://www.contributor-covenant.org/).
